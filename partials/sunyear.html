This example was suggested by Luke Franci on his website D3 for Mere Mortals. It is a custom visualization which cannot be produced by any commercial charting package. It shows the daylight hours for the selected city for each day throughout the current year. Each axis makes use of a D3 time scale. The example connects to two remote data sources: the autocomplete API provided by the Weather Underground (also used in the weather example) and the Google Timezone API, which provides the hour offset from GMT for the selected city based on its latitude and longitude. The example also uses a sunrise/sunset algorithm provided by a Web API belonging to this website.

There are 3 dashed lines running through the visualization. The two faint gray dashed lines represent the dates of the spring anf fall equinoxes. The solid black dashed line represents today's date. The orange circle on today's date line (if present) marks the time of today's sunrise. The yellow circle on today's date line marks today's midday, the time when the sun is directly overhead. (It is hard to see against the background of sunlight.) The red circle on today's date line (if present) marks the time of today's sunset. Times are not adjusted for daylight savings time.

Cities to try:

  1. Alta, Norway in the land of the midnight sun, north of the arctic circle
  2. Quito, Ecuador close to the equator
  3. Sydney, Australia in the southern hemisphere

This same visualization is used in the Sunclock website. The home page of the sunclock website shows the so-called sunrise/sunset terminator which shows which parts of the earth are experiencing daytime and which parts are experiencing nighttime at any given time.
